Guía Paso a Paso: Creación del Gestor de Recetas en Angular

Esta guía detalla minuciosamente todo el proceso realizado para construir la aplicación GourmetApp, un gestor de recetas culinarias desarrollado en Angular (versión moderna standalone) con persistencia en localStorage.

⚠️ RÚBRICA ACADÉMICA APLICADA: De acuerdo a los requerimientos indicados, el proyecto se ha configurado utilizando el formato clásico de nombrado de archivos de Angular (ej. lista-recetas.component.ts) en lugar de la guía moderna reducida del 2025. Además, la interfaz se estructuró con un diseño centrado de ancho fijo de 1100px para pantallas de escritorio, omitiendo el comportamiento fluido móvil a petición expresa.

📋 Requisitos Previos

  • Node.js instalado (versión utilizada en el desarrollo: v24.14.1).
  • npm instalado (versión utilizada en el desarrollo: 11.11.0).

🛠️ Paso 1: Inicialización del Proyecto Angular

Para iniciar el proyecto en la carpeta de trabajo actual, se ejecutó el siguiente comando desde la consola de terminal:

npx -y @angular/cli@latest new recetas-app --directory ./ --routing true --style css --ssr false --standalone true --file-name-style-guide 2016 --defaults --force

Explicación de las banderas del comando:

--directory ./
Especifica que el espacio de trabajo de Angular debe crearse directamente en la carpeta actual, evitando crear una subcarpeta adicional y manteniendo ordenados los archivos.
--routing true
Configura automáticamente el archivo src/app/app.routes.ts para permitir la navegación de una sola página (SPA).
--style css
Define el uso de hojas de estilo CSS estándar para dar formato personalizado a la aplicación.
--ssr false
Desactiva Server-Side Rendering (SSR). Esto es ideal para una entrega académica simple basada enteramente en el navegador del cliente.
--standalone true
Utiliza el estándar moderno de Angular basado en componentes autónomos (Standalone Components), evitando tener que registrar componentes en ficheros pesados AppModule.
--file-name-style-guide 2016
Fuerza a la CLI a generar archivos con la terminación tradicional del tipo de elemento (ej: app.component.ts y lista-recetas.component.ts), lo cual suele ser la convención de evaluación docente.
--defaults --force
Deshabilita las preguntas en consola para que se inicialice con configuraciones seguras por defecto, forzando la creación en el directorio de trabajo actual.

🗂️ Paso 2: Creación del Modelo de Datos Receta

Se estructuró una interfaz TypeScript que define los campos obligatorios y opcionales para cada receta culinaria.

Archivo creado: src/app/models/receta.model.ts

export interface Receta {
  id: string; // ID único generado dinámicamente
  nombre: string; // Título de la receta
  descripcion: string; // Descripción breve o introducción
  ingredientes: string[]; // Lista de ingredientes
  instrucciones: string[]; // Lista de pasos de preparación
  tiempoPreparacion: number; // Duración en minutos
  categoria: string; // Desayuno, Entrada, Almuerzo, Cena, Postre, etc.
  esFavorita: boolean; // Estado de favorito
  imagenUrl?: string; // Enlace web de imagen del plato (opcional)
}

💾 Paso 3: Servicio de Datos: RecetaService

El servicio centraliza todas las operaciones de almacenamiento y persistencia mediante el navegador.

Archivo creado: src/app/services/receta.service.ts

  • Mock Data inicial: Si el programa se abre por primera vez y el localStorage está vacío, se insertan automáticamente 3 recetas de demostración muy atractivas (Tacos al Pastor, Tarta de Manzana y Ensalada César).
  • Persistencia Local: Mediante localStorage.setItem() y localStorage.getItem() los datos de las recetas se guardan de forma permanente en el navegador del usuario y no se pierden al recargar la pestaña.
  • CRUD Completo: Proporciona métodos para listar (getRecetas()), buscar por ID (getRecetaById()), agregar nuevas recetas (agregarReceta()), editar registros existentes (actualizarReceta()), eliminar recetas (eliminarReceta()) y cambiar el estado favorito (toggleFavorito()).

🧱 Paso 4: Generación de Componentes

Para construir la interfaz y organizar el código modularmente de acuerdo a los requisitos, se ejecutaron los comandos de generación de componentes:

npx ng g c lista-recetas --skip-tests
npx ng g c form-receta --skip-tests
npx ng g c detalle-receta --skip-tests

(El parámetro --skip-tests evita la creación de archivos de pruebas unitarias spec.ts, agilizando el mantenimiento del código).

🚦 Paso 5: Configuración de Rutas (Routing)

Se modificó el archivo src/app/app.routes.ts para mapear la navegación entre secciones:

import { Routes } from '@angular/router';
import { ListaRecetasComponent } from './lista-recetas/lista-recetas.component';
import { FormRecetaComponent } from './form-receta/form-receta.component';

export const routes: Routes = [
  { path: '', redirectTo: 'inicio', pathMatch: 'full' },
  { path: 'inicio', component: ListaRecetasComponent },
  { path: 'agregar', component: FormRecetaComponent },
  { path: 'editar/:id', component: FormRecetaComponent },
  { path: 'favoritos', component: ListaRecetasComponent },
  { path: '**', redirectTo: 'inicio' }
];

💻 Paso 6: Lógica y Plantillas de los Componentes

1. Componente lista-recetas

Este componente se encarga de renderizar la cuadrícula de recetas culinarias.

  • Uso de directivas: Emplea *ngFor para iterar sobre la lista de recetas filtradas y mostrarlas en tarjetas. Utiliza *ngIf para renderizar vistas vacías cuando no hay resultados de búsqueda o no se han marcado recetas favoritas.
  • Búsqueda Integrada: A través de un enlace de doble vía (ngModel), detecta el texto en la barra de búsqueda y filtra al instante por nombre, categoría o ingredientes.
  • Ruta compartida: Reutiliza la misma plantilla para las rutas de Inicio y Favoritos, alternando los filtros en base a si el URL del navegador incluye la palabra "favoritos".

2. Componente form-receta

Formulario para registrar o editar información culinaria.

  • Formulario Dinámico: Evalúa la ruta para detectar si tiene una variable id. Si existe, cambia a modo "Edición" y precarga los campos con los datos de la receta para modificarlos. Si no, se inicializa como un formulario de "Creación".
  • Área de Texto Inteligente: Para facilitar la escritura del usuario, las listas de ingredientes e instrucciones se ingresan en cuadros de texto multilínea (un elemento por línea). Al guardar, el componente los formatea a un arreglo de cadenas string[] mediante funciones split('\n') y trim().

3. Componente detalle-receta

Ventana emergente (modal) de preparación.

  • Superposición Modal: Se abre directamente sobre la pantalla actual sin interrumpir la experiencia de navegación del usuario.
  • Función Premium: Permite tachar ingredientes interactivos. Al pulsar un ingrediente, cambia su estilo visual con una línea que lo tacha, ayudando al cocinero a controlar sus preparativos e ingredientes reunidos.

🎨 Paso 7: Diseño y Estilos de Escritorio Fijos

  • App Layout: Se estructuró un cabezal fijo (Navbar) con enlaces dinámicos para navegar a "Inicio", "Agregar receta" y "Favoritos", aplicando estados activos visuales de color naranja cálido.
  • Ancho Fijo Centrado: Siguiendo el requerimiento de omitir comportamiento responsivo, el contenido se estructuró con un ancho fijo de 1100px centrado horizontalmente en la pantalla (max-width: 1100px; margin: 0 auto;).
  • Tipografía Global: Se aplicó la fuente moderna Poppins en todo el proyecto a través de su importación global en src/styles.css.

🚀 Paso 8: Cómo Ejecutar y Probar el Proyecto

  1. Abre la consola de comandos en el directorio del proyecto (c:\CYNTHIA\recetas).
  2. Ejecuta el servidor de desarrollo local: 
    npm run start
    (Esto es el equivalente directo a ejecutar npx ng serve).
  3. Abre un navegador de internet (Chrome, Edge o Firefox) e ingresa a la siguiente URL: 
    http://localhost:4200
  4. Podrás ver el recetario funcionando completamente con recetas precargadas y persistencia activa.
